# 製品仕様書: 進捗・正答率トラッキング

## プロダクト概要
学習者（小5・6年生）とその保護者が「どの単元をどれだけ解いたか」「正答率は何%か」を一目で把握できる機能。localStorageに蓄積し、将来のAI分析・復習リスト機能の基盤にもなる。

## 機能一覧
| # | 機能名 | 概要 | 優先度 | スプリント |
|---|--------|------|--------|-----------|
| 1 | 正誤記録 | 演習問題の正誤をカテゴリ別にlocalStorageへ保存 | Must | 1 |
| 2 | 進捗ダッシュボード | ホーム画面またはサイドに進捗サマリーを表示 | Must | 1 |
| 3 | カテゴリ別正答率表示 | 単元一覧画面に正答率バッジを追加 | Must | 1 |
| 4 | 学習ストリーク強化 | 連続正解ストリークを既存ゲーム機能と統合 | Should | 2 |
| 5 | リセット機能 | 進捗データをリセットできる設定画面 | Should | 2 |

## スプリント計画

### Sprint 1: 記録 + 表示の基盤
- 機能: 正誤記録・進捗ダッシュボード・カテゴリ別正答率バッジ
- 完了条件:
  - [ ] ユーザーが演習問題に正解/不正解するたびにlocalStorageへ記録される
  - [ ] 記録データは `vista_stats` キーにカテゴリ別JSON形式で保存される
  - [ ] コース選択画面（home-screen）に「今日の進捗」サマリーが表示される（解いた問題数・全体正答率）
  - [ ] 単元一覧画面の各カテゴリカードに正答率バッジが表示される（例: 85% / 未挑戦）
  - [ ] 既存の `checkAnswer()` 関数と連携して自動記録される（ユーザーの手間ゼロ）

### Sprint 2: ストリーク強化 + リセット
- 機能: 連続正解強化・リセット設定
- 完了条件:
  - [ ] ユーザーが設定画面から学習データをリセットできる
  - [ ] リセット前に確認ダイアログが表示される
  - [ ] ストリークが既存の星バッジと連動して表示される

## 制約・前提条件
- 1ファイル構成（index.html）を維持する
- localStorageのみ使用（サーバー不要）
- 既存の `checkAnswer()` / `addStar()` 関数を拡張する形で実装
- データ構造は将来のClaude API分析に渡せる形にする

## データ構造（仮定）
```json
{
  "vista_stats": {
    "g5_exam": {
      "つるかめ算": { "correct": 3, "total": 5 },
      "比と割合": { "correct": 7, "total": 8 }
    },
    "g6_exam": {
      "速さ": { "correct": 2, "total": 3 }
    }
  },
  "vista_last_study": "2026-06-27"
}
```
