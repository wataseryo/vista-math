# 製品仕様書: VISTA 次フェーズ

## プロダクト概要

「ただの問題集」から「苦手を見える化して復習できるアプリ」への進化。
対象は中学受験を目指す小学5・6年生と、その保護者・塾講師。
「導入コンテンツ」「弱点復習」「簡易レポート」の3本柱を追加する。

---

## 機能一覧

| # | 機能名 | 概要 | 優先度 | スプリント |
|---|--------|------|--------|-----------|
| 1 | カテゴリ・unit分類の整理 | mainCategory / lessonUnit の揺れを修正し、食塩水・旅人算等を正しい親カテゴリへ整理 | Must | 1 |
| 2 | 導入コンテンツ（3カテゴリ） | 割合・比 / 速さ / 特殊算文章題 の各トピック画面に初学者向け導入セクションを追加 | Must | 2 |
| 3 | 弱点復習画面 | localStorageのデータから苦手単元・問題を分析して表示し、復習へ誘導する | Must | 3 |
| 4 | 保護者・塾向け簡易レポート | ログインなしでlocalStorageのデータを可視化し、一言コメントを自動生成 | Should | 4 |

## スプリント計画

### Sprint 1: カテゴリ・unit分類の整理

**目的**: データの整合性を整え、後続スプリントの分析・表示が正確に動くようにする。

**現状の確認事項**（コード調査済み）:
- `data/problems.js` の各問題は `mainCategory` と `lessonUnit` を持つ
- 現状の `mainCategory` 値: `'ratio'`, `'speed'`, `'word_problems'`, `'solid_geometry'`
- 食塩水は `mainCategory:'ratio'`, `lessonUnit:'食塩水'` として定義済み → 正しい
- 旅人算・通過算・流水算は `mainCategory:'speed'` として定義済み → 正しい
- つるかめ算・和差算・過不足算・植木算・年齢算は `mainCategory:'word_problems'` として定義済み → 正しい
- `categories.js` の `EXAM_UNITS_G6` に `u6-salt`（食塩水専用エントリ）が独立で存在する

**問題点**:
- `EXAM_UNITS_G6` の `u6-salt` が `num: '③+'` という独立した並び方をしており、カテゴリ`categories`の定義（8分野）では食塩水は `ratio` の傘下
- `PRACTICE_CATS` の `pc-ratio` には食塩水も含まれているが、表示上の説明文（`sub`）に食塩水の記載がない
- `EXAM_UNITS` (旧来の `EXAM_UNITS` 配列) に `category:'割合'` として `unit-ratio` が存在し、`category` フィールドの表記が新旧混在している

**完了条件**:
- [ ] `categories.js` の `PRACTICE_CATS` の `pc-ratio` の `sub` に「食塩水」が明記される
- [ ] `EXAM_UNITS_G6` の `u6-salt` が `u6-ratio` の派生であることがコメントで明記され、UIで「割合・比」グループの一部として扱われる
- [ ] `data/problems.js` 内で `mainCategory` の値が以下の4つのみになる（確認のみ、修正があれば実施）:
  - `'ratio'`（割合・比・食塩水・売買損益・仕事算・ニュートン算を含む）
  - `'speed'`（旅人算・通過算・流水算・時計算・ダイヤグラム・速さと比を含む）
  - `'word_problems'`（和差算・つるかめ算・過不足算・植木算・年齢算・平均算・消去算・分配算・集合算を含む）
  - `'solid_geometry'`（水槽・立体図形を含む）
- [ ] `CATEGORIES` 配列（8分野 正規ID）と実際の `mainCategory` 値のマッピングが `categories.js` のコメントに整理される
- [ ] 既存の localStorage (`vista_stats`, `vista_history`) のキー構造が変わらない
- [ ] 既存UIの表示・動作が壊れない（トピック学習・演習問題・難問チャレンジすべて）

### Sprint 2: 導入コンテンツを追加

**目的**: 各カテゴリのトピック学習に入る前に「この単元で何を学ぶか」を示し、初学者が迷わず学習を始められるようにする。

**対象カテゴリ**:
1. 割合・比（`u6-ratio` トピック画面）
2. 速さ（`u6-speed` トピック画面）
3. 特殊算・文章題（`u6-special` トピック画面）

**各導入コンテンツの構成**（7項目）:
1. **この単元で学ぶこと** — 1〜2文の概要
2. **入試でどう出るか** — 典型的な出題パターン（箇条書き2〜3件）
3. **問題文の見分け方** — キーワード・フレーズのヒント
4. **使う図・考え方** — 面積図・線分図・ダイヤグラム等（テキストで説明、簡易ASCII図でもよい）
5. **よくあるミス** — 2〜3件
6. **基本例題** — 1問（問題文・解答・解説）。難問ではなく入門レベル
7. **演習問題への導線** — 「演習問題を解く」ボタン（該当カテゴリへ直接ジャンプ）

**完了条件**:
- [ ] `u6-ratio` のトピック画面に導入セクションが表示される
- [ ] `u6-speed` のトピック画面に導入セクションが表示される
- [ ] `u6-special` のトピック画面に導入セクションが表示される
- [ ] 各導入コンテンツに7項目すべてが含まれる
- [ ] 基本例題の答えは整数のみ（小数・分数の答えは避ける）
- [ ] 「演習問題を解く」ボタンを押すと、該当カテゴリの演習問題一覧へ遷移する
- [ ] 導入セクションは既存のトピック問題リストの「上」に配置される（スクロールで問題へ進める）
- [ ] 5年生向けトピック画面には導入コンテンツを追加しない（6年生のみ。仮定として明記）
- [ ] 既存のトピック問題（解説・回答機能）が壊れない

### Sprint 3: 弱点復習画面

**目的**: `vista_stats`（カテゴリ別正答データ）と `vista_history`（問題別試行履歴）を分析し、ユーザーが「自分の苦手」をひと目で把握して復習に進める画面を作る。

**画面の配置**:
- 受験コース（`exam-mode-screen`）の学習モード選択カードに「弱点復習」カードを追加する

**表示内容**（5つのセクション）:

1. **苦手カテゴリ TOP3**
   - `vista_stats` の `g{grade}_exam_{catId}` キーから正答率を計算
   - 正答率が低い順に最大3カテゴリを表示
   - 正答率・解答済み問題数を表示
   - 未挑戦（total=0）のカテゴリは除外

2. **苦手unit TOP3**
   - `vista_history` の問題ID → `PRACTICE_PROBLEMS` の `lessonUnit` を逆引き
   - lessonUnit別に正答率を集計し、低い順に最大3件表示
   - 3回以上挑戦したunitのみ対象（少数サンプルを除外するため）

3. **最近間違えた問題**（最大5件）
   - `vista_history` で最後の試行が不正解（`false`）の問題IDを収集
   - `PRACTICE_PROBLEMS` から問題タイトル・カテゴリを取得して表示

4. **おすすめ復習単元**（1〜2件）
   - 苦手カテゴリ TOP1 の lessonUnit で正答率が最も低いunitを推薦
   - データが少ない場合は「まずはこの分野に挑戦」として未挑戦カテゴリを表示

5. **復習ボタン**
   - 苦手カテゴリ・おすすめ単元の横に「この単元を復習する」ボタン
   - 押すと対応する演習問題カテゴリへ直接遷移

**データが少ない場合の表示**:
- 解答数が5問未満の場合は「まずは問題を解いてみよう！」メッセージを表示
- 各セクションでデータが存在しない場合はそのセクションを非表示

**完了条件**:
- [ ] 受験コースの学習モード選択画面に「弱点復習」カードが表示される
- [ ] 弱点復習画面に5つのセクションが表示される
- [ ] 苦手カテゴリ TOP3 が正答率順（低い順）で正しく表示される
- [ ] 苦手unit TOP3 が表示される（3回以上挑戦したunitのみ）
- [ ] 最近間違えた問題が最大5件表示される
- [ ] 「この単元を復習する」ボタンを押すと演習問題へ遷移する
- [ ] データが5問未満の場合は専用メッセージが表示される
- [ ] Claude APIや外部サービスを使わず、localStorageのデータのみで動作する
- [ ] 既存の localStorage キー（`vista_stats`, `vista_history`）の構造を変更しない

### Sprint 4: 保護者・塾向け簡易レポート画面

**目的**: ログイン不要で、localStorageのデータから学習状況を可視化したレポートを表示する。保護者・塾講師が子どもの進捗を把握し、サポートに活かせるようにする。

**画面の配置**:
- 受験コースの学習モード選択画面に「レポートを見る」ボタン（カードではなくテキストリンク形式）を追加する

**表示内容**（6つのセクション）:

1. **全体サマリー**
   - 解いた問題数（total）
   - 全体正答率（%）
   - 最終学習日（`vista_history` のタイムスタンプ。仮定: 本スプリントでタイムスタンプ記録を追加する）

   仮定: `recordProblemAttempt` に `Date.now()` のタイムスタンプを追加して `vista_history` に保存する方式に変更する。既存の試行履歴（`true`/`false` の配列）の構造は維持しつつ、新規記録からタイムスタンプを付与する。

2. **カテゴリ別正答率**
   - `PRACTICE_CATS` の8分野それぞれの正答率をバー表示
   - 未挑戦分野は「未挑戦」と表示

3. **苦手単元 TOP3**
   - Sprint 3 と同じロジック（正答率が低い lessonUnit 上位3件）

4. **最近間違えた問題**（最大3件）
   - 問題タイトル・カテゴリ名を表示

5. **次に復習すべき単元**（1件）
   - 苦手単元 TOP1 を表示

6. **保護者向け一言コメント**（ルールベース自動生成）
   - 以下のルールで1文を自動生成:
     - 全体正答率 80%以上 → 「全体的によく解けています。苦手な〇〇をさらに強化しましょう。」
     - 全体正答率 50〜79% → 「着実に力がついています。〇〇の復習を集中的にやってみましょう。」
     - 全体正答率 50%未満 → 「〇〇を中心に、基本問題から丁寧に取り組みましょう。」
     - 解答数が5問未満 → 「まだ学習が始まったばかりです。まずは気軽に問題を解いてみましょう。」
   - 〇〇には苦手カテゴリ名 TOP1 を自動挿入

**完了条件**:
- [ ] 受験コース学習モード選択画面に「レポートを見る」リンクが表示される
- [ ] レポート画面に6つのセクションがすべて表示される
- [ ] カテゴリ別正答率がバーグラフで視覚的に表示される
- [ ] 保護者向け一言コメントが正答率に応じた4パターンのいずれかで表示される
- [ ] 一言コメントに苦手カテゴリ名が正しく挿入される
- [ ] 解答数が5問未満の場合は専用メッセージが表示される
- [ ] ログインなし・Claude APIなしで動作する
- [ ] Sprint 3 で追加したタイムスタンプ記録と連携して最終学習日が表示される（Sprint 3 完了後に動作確認）
